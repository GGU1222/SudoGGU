# 숨꾸찾 스도꾸 프로젝트 설명

## 1. 게임 개요
- **장르**: 스도쿠류 논리 퍼즐 (LinkedIn "Queens" 유형과 유사)
- **컨셉**: N×N 격자에서 각 색상 영역·행·열에 토끼가 정확히 한 마리씩 존재하도록, 클릭(X 표시)·더블클릭(토끼 표시)·드래그(여러 칸 X 표시)로 소거법을 통해 위치를 추론하는 게임
- **코드네임**: "sudoGGU" (리포지토리명과 통일), 실제 인게임 표시명은 "숨꾸찾 스도꾸"

## 2. 기술 스택
| 항목 | 내용 |
|---|---|
| 구현 방식 | 순수 HTML / CSS / Vanilla JavaScript, 빌드 도구·프레임워크 없음 |
| 배포 | GitHub Pages (리포 최상위 `index.html`을 그대로 서빙) |
| 대상 | 데스크톱·모바일 브라우저 (터치 입력 포함), 라이트/다크 모드 대응 |
| 아트 리소스 | placeholder (색상 사각형 + 이모지) |

## 3. 게임 규칙
1. 격자는 **9×9 고정**, 색상 개수는 항상 격자 크기와 동일(9개)
   - 같은 색상 영역은 서로 최소 한 칸 이상 붙어(4방향 연결) 있어야 하며, 1칸짜리 영역은 생기지 않음
2. 정답(토끼 위치) 제약
   - 색상 영역당 토끼 1마리
   - 행(row)당 1마리, 열(column)당 1마리
   - 인접한 칸(상하좌우 + 대각선 포함 8방향)에는 토끼가 동시에 존재할 수 없음
3. **입력 방식**
   - 클릭(또는 드래그): 즉시 X 표시 (다시 클릭하면 빈칸으로 토글)
   - 같은 칸을 300ms 이내에 한 번 더 클릭: 방금 찍은 X를 토끼 표시로 승격
   - 클릭한 채로 드래그: 지나간 모든 칸을 X로 일괄 표시(토글 없이 항상 X)
4. 실제로 없는 칸에 토끼 표시 → 목숨 1 차감(최대 3목숨), 표시는 검붉은색 X로 바뀜(일반 X와 색상으로 구분). 이미 표시된 칸을 다시 표시해도 중복 차감 없음
5. 실제로 있는 칸에 X 표시 → 페널티 없음
6. 정답으로 맞춘(실제 토끼가 있는) 칸은 그 이후 클릭/더블클릭/드래그로 다시 바뀌지 않도록 잠김
7. 목숨 3회 소진(게임 오버) 또는 토끼를 모두 찾음(클리어) → 2초 후 다음 퍼즐 자동 생성 (별도 버튼 없음)
8. 화면 상단에 게임 방법 안내 버튼(❓), 남은 목숨, 찾은 토끼 수, 새 게임 버튼이 나란히 배치됨. score, 힌트 기능은 구현하지 않음
9. 마크/지우기 동작마다 짧은 진동 피드백 (Vibration API 지원 기기 한정 — iOS Safari는 API 자체가 없어 무동작)

## 4. 퍼즐 생성 알고리즘
1. **정답 배치**: 각 행마다 한 열을 선택하되 열 중복 금지 + 8방향 인접 금지 조건을 만족하도록 백트래킹으로 생성
2. **색상 영역 성장**: 정답 토끼 좌표를 각 색상의 씨앗으로 삼아 나머지 칸을 한 칸씩 무작위로 확장하되, 배정 전에 솔버로 유일해 유지 여부를 확인 후 커밋
3. **유일해 검증**: 완전탐색 솔버로 해가 정확히 1개인지 확인 → 아니면 1번부터 재시도
4. 색상 개수를 격자 크기와 다르게 하거나 격자 크기를 9보다 키우면 유일해 퍼즐 생성 성공률이 사실상 0%가 되는 것으로 확인되어, 현재 이 두 값은 고정되어 있음

## 5. 코드 구조
```
index.html                     # 진입점 (리포 최상위) — GitHub Pages가 그대로 서빙
web/
 ├─ css/
 │   └─ style.css
 └─ js/
     ├─ core/                 # 순수 로직, DOM 비의존 (Node에서도 그대로 실행/검증 가능)
     │   ├─ gameRules.js       # 8방향 인접 판정
     │   ├─ board.js           # 보드 상태(colorOf/hasRabbit/mark) + 클리어 판정
     │   ├─ puzzleSolver.js     # 유일해 검증용 백트래킹 솔버
     │   └─ puzzleGenerator.js  # 퍼즐 생성 파이프라인 구현
     ├─ game/
     │   └─ gameManager.js      # 목숨 관리, 표시 판정(잠김 포함), 게임오버/클리어, 이벤트 발행
     └─ ui/
         ├─ boardInputController.js  # 클릭/더블클릭/드래그 제스처 인식 → GameManager 호출
         ├─ boardView.js        # Board 데이터를 격자 DOM으로 렌더링
         └─ livesView.js        # 목숨 표시
```
- `js/core/`는 DOM/브라우저 API에 의존하지 않는 순수 JS라, Node.js에서도 그대로 실행해 알고리즘을 검증할 수 있음
- `web/js`, `web/css` 소스를 수정하면 최상위 `index.html`이 바로 그 파일들을 상대경로로 참조하고 있어 빌드 없이 즉시 반영됨

## 6. 현재 범위
- 완료: 퍼즐 생성/검증, 격자 UI 렌더링, 클릭/더블클릭/드래그 입력, 목숨/게임오버/클리어, 게임 방법 안내 팝업, 모바일 터치 대응, 라이트/다크 모드
- 미구현: score, 힌트 기능, 단계별 난이도 조절
- 아트는 placeholder(색상 사각형 + 🐰 이모지 / X 막대 도형)로 진행 중이며, 영역 색상 팔레트는 CIE Lab 기준 색상 간 최소 인지 거리(Delta-E)를 계산해 검증됨

## 7. 알려진 제한사항
1. **모바일 앱 패키징**: 필요 시 PWA(홈 화면 추가) 또는 Capacitor 같은 웹뷰 래퍼로 앱스토어 배포 가능하나 아직 실제 검증은 안 함
2. **격자 크기 확장(10×10 이상)**: 현재 생성 알고리즘으로는 사실상 불가능함이 확인됨. 필요해지면 CP-SAT/ILP 솔버 등 다른 접근이 필요
